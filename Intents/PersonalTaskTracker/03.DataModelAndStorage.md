# 3. DataModelAndStorage

**Goal**: Define the schema for boards, columns, and cards (incl. color, due date) and implement local persistence.
**Est.**: 2h
**Dependencies**: 02

## Steps
- [x] Define data types for Board, Column, Card (title, description, color, dueDate, position)
- [x] Choose storage mechanism (e.g. SQLite or a JSON file) consistent with the offline requirement
- [x] Implement read/write functions (load all boards, save board, save card, etc.)
- [x] Write a smoke test: create a board, save, reload app, confirm data persists

## Definition of Done
- [x] Data survives an app restart — `better-sqlite3` file at `app.getPath('userData')/taskapp.db`; smoke test reopens the same file and confirms data persists
- [x] CRUD-ready storage API available to the UI layer — `electron/db.js` store + IPC handlers in `electron/main.js` + `window.taskApi` bridge in `electron/preload.js` (typed via `src/shared/taskApi.d.ts`)
