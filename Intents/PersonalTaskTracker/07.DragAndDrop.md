# 7. DragAndDrop

**Goal**: Drag cards between columns and reorder within a column; persist the new position.
**Est.**: 2h
**Dependencies**: 06

## Steps
- [x] Integrate a drag-and-drop library compatible with the chosen stack
- [x] Support dragging a card to a different column
- [x] Support reordering within the same column
- [x] Persist the new column/position to storage on drop

## Definition of Done
- [x] Cards can be dragged between and within columns — `@dnd-kit/core` + `@dnd-kit/sortable`, multi-container pattern (`DndContext` + per-column `useDroppable` + `SortableContext`)
- [x] New position survives an app restart — `reorderColumn(columnId, cardIds)` rewrites `column_id`/`position` in one transaction; covered by `npm run smoke:db`

## Outcome (fill after Iterate)
- **Actual Time**:
- **Result**: Lifted card state to the board level (`src/state/useBoardCards.ts`, replacing the per-column `useCards` hook) so a card can move between columns' local arrays during drag. Added `electron/db.js#listCardsByBoard`/`reorderColumn`, IPC handlers, and `window.taskApi.cards.listByBoard`/`reorderColumn`. `KanbanBoard.tsx` now wraps columns in `DndContext` with a `PointerSensor` (8px activation distance, so clicks-to-edit still work), live cross-column preview in `onDragOver`, and persists the final order of affected column(s) in `onDragEnd` via `commitOrder`. `CardItem.tsx` uses `useSortable`; `DragOverlay` shows a floating copy of the card while dragging. `npm run smoke:db` extended to cover `reorderColumn` across a restart — passes. `npm run build` succeeds.
- **Follow-ups**: Same sandbox GUI limitation as prior UI intents — drag-and-drop is the highest-value thing to manually verify on your machine: drag a card within a column (reorders), drag a card to a different column (moves + reorders), restart the app and confirm both stuck.
