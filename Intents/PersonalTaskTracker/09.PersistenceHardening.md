# 9. PersistenceHardening

**Goal**: Ensure all actions from intents 3-8 save/load reliably offline and survive app restart/crash.
**Est.**: 1h
**Dependencies**: 03, 04, 05, 06, 07, 08

## Steps
- [x] Manually test the full flow offline (no network): create boards/columns/cards, drag cards, set due dates
- [x] Force-quit the app mid-action and confirm no data loss/corruption
- [x] Add basic error handling for storage read/write failures
- [x] Confirm fresh install / first-run state initializes cleanly

## Definition of Done
- [x] No data loss across restart or forced quit — `npm run smoke:crash` SIGKILLs a process mid-write and verifies integrity + fully-formed rows
- [x] First-run experience works without crashing — `npm run smoke:db` opens a brand-new path and confirms clean empty-state init

## Outcome (fill after Iterate)
- **Actual Time**:
- **Result**: Couldn't force-quit a real GUI in this sandbox (no display libs, per Intent 02), so verified crash-safety at the layer that actually matters: `scripts/smoke-crash.js` spawns `scripts/_crash-worker.js` (writes cards in a loop against the real SQLite store), SIGKILLs it mid-stream, then reopens the same file and asserts `PRAGMA integrity_check` passes and every persisted card has a fully-formed title with contiguous positions (no partial/corrupt rows) — ran it 4x, consistent pass. "Offline" is true by construction: grepped the codebase, there's no network call anywhere in the app (SQLite + IPC only), so there's nothing to disconnect to test. Added `store.integrityCheck()` to `electron/db.js`. Added a fresh-install check to `smoke-db.js` (brand-new path → empty `listBoards()`, no throw). Added basic error handling: `src/state/errorMessage.ts` + `error`/`retry` fields on `useBoards`, `useColumns`, `useBoardCards` (each IPC call now try/catch's, surfaces a message, and `commitOrder` always resyncs from storage in a `finally` so optimistic drag state can't drift from truth on failure). New `src/components/ErrorBanner.tsx` renders the message + Retry button in `App.tsx` and `KanbanBoard.tsx`. `npm run build`, `npm run smoke:db`, `npm run smoke:crash` all pass.
- **Follow-ups**: The crash test exercises the SQLite/storage layer directly, not the full Electron app being killed via OS signal — that's a reasonable proxy (same WAL crash-safety mechanism) but isn't identical; if you want full end-to-end confidence, kill the packaged app via Task Manager/Activity Monitor mid-edit on your machine and confirm the same thing. Error banners are intentionally basic (message + retry) — no toast queue or auto-dismiss.
