# 3. ReorderColumnsBackend

**Goal**: Add IPC + storage support to persist a new order for columns within a board, mirroring the existing `cards:reorderColumn` pattern.
**Est.**: 1h
**Dependencies**: none

## Steps
- [x] In `electron/db.js`, add `store.reorderColumns(columnIds)`: a transaction that sets `position = index` for each id in `columnIds`, in order (mirror `reorderColumn` at db.js:152, but without the `column_id`/board reassignment since columns don't move between boards in this pass).
- [x] In `electron/main.js`, add `ipcMain.handle("columns:reorder", (_e, columnIds) => store.reorderColumns(columnIds))`.
- [x] In `electron/preload.js`, expose `columns.reorder: (columnIds) => ipcRenderer.invoke("columns:reorder", columnIds)`.
- [x] In `src/shared/taskApi.d.ts`, add `reorder(columnIds: number[]): Promise<void>;` to the `columns` interface.
- [x] Confirm `columns.list(boardId)` already returns rows ordered by `position`; add `ORDER BY position` if missing.

## Definition of Done
- [x] Calling `window.taskApi.columns.reorder([id2, id1, id3])` persists that order to SQLite.
- [x] `columns.list(boardId)` returns columns in `position` order after a reorder call.
- [x] `npm run build` (or equivalent typecheck) passes with no type errors.
