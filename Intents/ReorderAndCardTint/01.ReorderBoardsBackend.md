# 1. ReorderBoardsBackend

**Goal**: Add IPC + storage support to persist a new order for boards, mirroring the existing `cards:reorderColumn` pattern.
**Est.**: 1h
**Dependencies**: none

## Steps
- [x] In `electron/db.js`, add `store.reorderBoards(boardIds)`: a transaction that sets `position = index` for each id in `boardIds`, in order (mirror `reorderColumn` at db.js:152).
- [x] In `electron/main.js`, add `ipcMain.handle("boards:reorder", (_e, boardIds) => store.reorderBoards(boardIds))`.
- [x] In `electron/preload.js`, expose `boards.reorder: (boardIds) => ipcRenderer.invoke("boards:reorder", boardIds)`.
- [x] In `src/shared/taskApi.d.ts`, add `reorder(boardIds: number[]): Promise<void>;` to the `boards` interface.
- [x] Confirm `boards.list()` already returns rows ordered by `position` (check the `SELECT` in db.js) — if not, add `ORDER BY position`.

## Definition of Done
- [x] Calling `window.taskApi.boards.reorder([id3, id1, id2])` persists that order to SQLite.
- [x] `boards.list()` returns boards in `position` order after a reorder call.
- [x] `npm run build` (or equivalent typecheck) passes with no type errors.
