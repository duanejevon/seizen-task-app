# 4. BackgroundImageUI

**Goal**: Render the chosen background behind the app with a faded overlay, and build the picker UI (gallery + browse button) wired to Intent 3's infra.
**Est.**: 2h
**Dependencies**: 02, 03

## Steps
- [ ] Render the active background image full-bleed behind the app shell, with a fixed/semi-transparent overlay so it's always faded regardless of image content
- [ ] Build a background picker UI: thumbnails for built-in presets (incl. the default) + a "Browse for image…" button
- [ ] Wire picker actions to the Intent 3 IPC (`setBackground`, `listBuiltInBackgrounds`)
- [ ] Confirm translucent surfaces from Intent 2 layer correctly over the real background image (not just the placeholder)
- [ ] Verify the chosen background persists across an app restart

## Definition of Done
- [ ] User can choose a background from the gallery or browse for a custom file; it renders faded behind the translucent UI
- [ ] Selection persists across restarts
- [ ] Legibility holds up across a few different test images (light, dark, busy/high-contrast)
