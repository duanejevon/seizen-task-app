# 5. HamburgerAppMenu

**Goal**: Hide Electron's native menu bar and add an in-app hamburger button/menu that currently houses the background picker entry point.
**Est.**: 1.5h
**Dependencies**: 04

## Steps
- [x] Set `Menu.setApplicationMenu(null)` (or equivalent) in `electron/main.js` to remove the native File/Edit/View/Window/Help bar
- [x] Add a hamburger icon button to the app shell
- [x] Build a simple dropdown/panel menu triggered by the hamburger button
- [x] Move the Intent 4 background picker entry point into this menu
- [x] Structure the menu component so future settings entries can be added without rework

## Definition of Done
- [x] Native menu bar no longer appears
- [x] Hamburger button opens an app menu containing the background picker
- [x] Board sidebar remains always visible (unaffected by the hamburger menu)

## Outcome (fill after Iterate)
- **Actual Time**:
- **Result**: `Menu.setApplicationMenu(null)` removes the native menu bar. Added `AppMenu.tsx`: a hamburger button + dropdown panel (click-outside and Escape both close it), housing `BackgroundPicker`. Hit and fixed a real bug along the way: `BackgroundPicker` previously called `useBackground()` internally, so the background only applied the first time the menu was opened (it wasn't mounted before that) — fixed by hoisting the hook to `App.tsx` (always mounted) and passing the result down as a prop through `BoardSwitcher` → `AppMenu` → `BackgroundPicker`. Verified live via Playwright: no native menu, background renders correctly on launch without ever opening the menu, panel opens/closes correctly including click-outside.
- **Follow-ups**: None — menu is structured as a single panel for now; additional settings entries (if any) can be added inside `.app-menu-panel` without restructuring.
