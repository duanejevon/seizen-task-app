# 3. BackgroundImageInfra

**Goal**: Main-process IPC support for selecting and storing a custom background image, independent of the UI that will consume it.
**Est.**: 2h
**Dependencies**: none (parallel track to 01/02)

## Steps
- [ ] Add a small persistence mechanism for app-level settings (e.g. a `settings` table in the existing SQLite store) to store the chosen background reference
- [ ] Add an Electron main-process handler using `dialog.showOpenDialog` to let the user browse for a local image file
- [ ] Copy the selected file into `app.getPath('userData')/backgrounds/` with a generated filename; store its reference in settings
- [ ] Expose `getBackground` / `setBackground` (browse) / `listBuiltInBackgrounds` via the `window.taskApi` IPC bridge
- [ ] Bundle the user-supplied default image (and any additional built-in presets) as importable assets from `src/assets/backgrounds/`
- [ ] Add a smoke test covering: setting a background, restarting the store, confirming the reference persists

## Definition of Done
- [ ] IPC round-trip works: browse → copy into userData → persisted reference → retrievable on next launch
- [ ] Built-in default image is bundled and selectable via the same API surface
- [ ] Smoke test passes
